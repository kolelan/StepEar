
## Разработка PWA «Слуховой тренажёр – Угадай ступень»

### Общая концепция
Нужно реализовать прогрессивное веб-приложение (PWA) для тренировки музыкального слуха. Приложение воспроизводит вступительную каденцию, затем аккорд одной из ступеней текущей гаммы. Пользователь угадывает номер ступени. После правильного ответа проигрывается «закрепляющая» последовательность аккордов. Реализовать адаптивную систему, статистику, пользовательские гаммы, экспорт статистики для ИИ-консультаций.

### 1. Звуковое ядро
- Использовать **Web Audio API**.
- По умолчанию все ноты звучат как **фортепиано** (можно синтезировать простыми волнами с огибающей или загрузить лёгкие сэмплы, но лучше – синтез для экономии веса PWA).
- **В будущем расширяемо**: пользователь может назначить для каждой ноты отдельный инструмент (например, до → фортепиано, ми → скрипка, соль → труба). В первой версии достаточно только фортепиано, но архитектура должна позволять легко добавить выбор инструментов.
- Аккорды играются **блоком** (все ноты одновременно) или **арпеджио** – на выбор пользователя (по умолчанию блок).
- Диапазон нот: малая октава, первая октава, вторая октава. Пользователь может выбрать, какие октавы использовать для построения гаммы (по умолчанию только первая октава). **Вне диапазона:** ступени, чей аккорд (каденция, вопрос, закрепление) не помещается в выбранные октавы, **не задаются** — при генерации вопросов и выборе ступеней для тренировки учитывать только допустимые ступени.
- **Голосоведение (voicing):** подбирать обращения и расположение голосов максимально близко к естественному пианистическому/хоровому поведению: общие тоны удерживать, соседние голоса по возможности вести секундой, избегать параллельных квинт и октав, разумные скачки в басу, склонности (ведущий тон к тонике, субдоминантовый звук к тонике) — в пределах диатоники текущей гаммы. Между последовательными аккордами одного упражнения сохранять преемственность регистра (не «перескакивать» все голоса на новую октаву без нужды).

### 2. Гаммы и аккорды ступеней
- **Стандартные мажорные гаммы** от всех 12 нот (C, C#, D, D#, E, F, F#, G, G#, A, A#, B).
- Каждая ступень мажорной гаммы образует трезвучие по терциям:  
  1 – мажорное, 2 – минорное, 3 – минорное, 4 – мажорное, 5 – мажорное, 6 – минорное, 7 – уменьшённое.
- **8-я ступень** – тоника на октаву выше, её аккорд полностью повторяет 1-ю ступень (только ноты выше на октаву).
- Пользователь может:
  - выбрать любую стандартную гамму;
  - выбрать «случайная гамма» – каждый вопрос новая случайная тональность;
  - создать **свою гамму** (набор нот, название, аккорды для каждой ступени – либо автоматически по терциям, либо вручную назначить ноты аккорда для каждой ступени);
  - выбрать **гармонический минор**, **мелодический минор** (вверх) как предустановленные.
- **Пользовательские аккорды ступеней**: интерфейс в виде таблицы из 7–8 строк. Для каждой ступени пользователь выбирает ноты (например, через виртуальную пианино или список). Сохранять как пресет гаммы.

### 3. Вступительная каденция
- Пользователь может выбрать **тип каденции** из списка (по умолчанию T–D–S–T, то есть тоника → доминанта → субдоминанта → тоника, аккордами):
  1. T–S–D–T
  2. T–D–S–T
  3. T–S–T–D–T
  4. T–D–T–S–T
  5. Только тонический аккорд (без каденции)
- Каждый аккорд каденции играется длительностью, зависящей от темпа (целая нота: **4/темп** секунд при темпе в bpm, например 1 с при 60 bpm). **Пауза между аккордами каденции – 0,5 с** (фиксированная).
- После каденции – **пауза 0,5 с**, затем **аккорд-вопрос** (та же длительность, что аккорды каденции).

### 4. Темп воспроизведения
- Пользователь может регулировать темп (ползунок или числовое поле) в диапазоне **40–160 bpm**.
- Темп влияет только на **длительность звучания** аккордов (каденция, аккорд-вопрос, закрепляющая последовательность).
- **Паузы фиксированы: 0,5 с** — между аккордами каденции, после каденции перед вопросом, между аккордами закрепляющей последовательности. От темпа не зависят.

### 5. Неверный ответ
- Каждый неверный выбор ступени **сразу засчитывается как отдельная ошибка** для адаптивной системы (правило 1) и в детальной истории попыток; на **процент упражнения** см. §9.
- На **том же вопросе** пользователь остаётся: закрепляющая последовательность **не** проигрывается, к следующему вопросу **не** переходим.
- **Показ правильной ступени после ошибки** (настройка упражнения или глобальная, по умолчанию **выкл.**):
  - **Вкл.:** после неверного выбора можно подсветить/показать правильную ступень (аккорд при этом не обязателен).
  - **Выкл.:** правильная ступень **не показывается** — ждём верного ввода пользователя.
- Доступны: повторный выбор ступени, **«Повторить с каденцией»** (каденция + аккорд-вопрос), «Только вопрос» (без каденции), **«Пропустить вопрос»** — вопрос засчитывается как **неуспешный** для процента упражнения; правильная ступень показывается только если включена та же настройка показа, иначе переход без подсказки.
- **Bigram (правило 2):** переход `prev → next` в матрицу ошибок попадает только если на вопрос `prev` в итоге был дан **верный** ответ; промежуточные ошибки на `prev` bigram не усиливают.

### 6. Последовательность после правильного ответа (закрепление)

**Принцип:**  
После угаданной ступени проигрывается «мелодическая линия» от этой ступени к тонике (или к верхней тонике).  
Направление движения зависит от положения ступени относительно тоники:

| Ступень | Движение | Последовательность аккордов |
|---------|-----------|-----------------------------|
| 1 | остаёмся | 1 |
| 2 | вниз | 2 → 1 |
| 3 | вниз | 3 → 2 → 1 |
| 4 | вниз | 4 → 3 → 2 → 1 |
| 5 | вверх | 5 → 6 → 7 → 8 (8 – тоника на октаву выше) |
| 6 | вверх | 6 → 7 → 8 |
| 7 | вверх | 7 → 8 |
| 8 | остаёмся | 8 |

**Пояснение на примере до мажора:**  
- Вопрос: 4-я ступень (F – субдоминанта) → после ответа играем **F → E → D → C** (вниз к тонике).  
- Вопрос: 5-я ступень (G – доминанта) → после ответа играем **G → A → B → C** (вверх к верхней тонике).  

Это не опечатка, а особенность тренажёра: так пользователь запоминает тяготение субдоминанты вниз, а доминанты – вверх.

- Длительность каждого аккорда в закреплении — как у каденции; **пауза между аккордами — 0,5 с** (фиксированная).

### 7. Режим тренировки
- Пользователь выбирает **какие ступени тренировать** (список чекбоксов 1–8). Генерируются вопросы только из выбранных ступеней.
- Количество вопросов в упражнении: любое число (например, 20). Пользователь вводит число.
- Пользователь может **сохранить упражнение** (настройки: гамма/случайная/пользовательская, тип каденции, темп, выбранные ступени, кол-во вопросов, **показывать правильную ступень после ошибки**, градации цветов статистики, включена ли адаптивная система) и дать ему имя. Сохранённые упражнения хранятся в IndexedDB (для зарегистрированных – синхронизация с сервером). Можно загрузить ранее сохранённое упражнение.
- Во время упражнения показывается **текущий прогресс** (вопрос N из M) и **процент упражнения** по правилу §9.

### 8. Адаптивная система (индексы вероятности)
- **Базовая вероятность** для каждой ступени (из выбранных пользователем) равномерная.
- **Правило 1 – слабые ступени:** система хранит количество правильных и неправильных ответов для каждой ступени за последние **K** ответов (K настраивается, по умолчанию 10). Для каждой ступени вычисляется `ошибка = (неправильно) / (всего ответов)` (если ответов меньше 3 – базовая вероятность не меняется). Индекс вероятности ступени `p_i = base_i * (1 + ошибка * 2)`. Затем нормализация до суммы 1.
- **Правило 2 – опасные последовательности (bigram):** хранится матрица `errors[prev_step][next_step]` – сколько раз пользователь ошибся на `next_step`, когда предыдущий вопрос был `prev_step` и на него был дан **правильный ответ**. Для каждого перехода `prev → next` увеличиваем вес `next` на `errors * 0.3`. Итоговая вероятность для ступени `next` = `p_i` (из правила 1) + бонус от bigram. Снова нормализация.
- **Адаптация применяется только если включена пользователем** (чекбокс «Адаптивные вопросы»). По умолчанию включена.
- После каждого ответа пересчитывать вероятности для следующего вопроса.

### 9. Статистика и отображение
- **Процент упражнения (сессии):** считается **по вопросам**, не по отдельным нажатиям. Вопрос засчитывается **верным**, если пользователь в итоге указал правильную ступень (в том числе **после одной или нескольких ошибок**). Вопрос **неверный**, если нажато «Пропустить вопрос» или упражнение прервано до верного ответа на этом вопросе. Формула: `(верных вопросов) / (завершённых вопросов) × 100%`.
- Отдельные ошибочные нажатия **не понижают** процент упражнения, если вопрос в конце отвечен верно; они учитываются в адаптивности (§8) и в детальном логе для экспорта.
- Для текущей сессии показывать:
  - общий процент упражнения (см. выше);
  - процент по каждой ступени отдельно по тому же правилу «верный вопрос» (если по ступени было хотя бы 2 вопроса).
- Пользователь выбирает **базу для статистики** – количество последних упражнений (например, 10). Упражнением считается одна завершённая сессия (например, из 20 вопросов). Статистика накапливается в истории упражнений.
- **Цветовая подсветка процента** (настраиваемая пользователем границы, по умолчанию):
  - 90–100% → зелёный
  - 80–89% → жёлтый
  - 70–79% → оранжевый
  - <70% → красный
- Пользователь может изменить эти пороги в настройках приложения (глобально или для конкретного упражнения).

### 10. Регистрация и локальное хранение
- **Без регистрации:** вся статистика, сохранённые упражнения, пользовательские гаммы хранятся в IndexedDB браузера.
- **С регистрацией:** создаётся аккаунт (email + пароль или OAuth). Данные синхронизируются с сервером (бэкенд на выбор – Firebase или простой REST). При входе – объединение локальных и серверных данных (пользователю предлагается выбор: заменить локальные на серверные, объединить или оставить локальные).
- PWA работает офлайн – все функции, кроме синхронизации и отправки статистики в DeepSeek, доступны.

### 11. Экспорт статистики для ИИ-консультаций (DeepSeek.com и др.)
- Кнопка «Экспорт статистики для ИИ». Формат – **CSV или JSON** (на выбор пользователя). Содержит:
  - Даты упражнений (или порядковые номера)
  - Для каждого упражнения: гамма, выбранные ступени, список ответов (ступень вопроса → ответ пользователя → правильность)
  - Агрегированные проценты за период, который пользователь выбирает (последние N дней, последние M упражнений).
- Дополнительная кнопка «Копировать промпт для DeepSeek» – формирует текст вида: *«Вот моя статистика тренировок слуха за последние 10 дней: [данные]. Какие у вас рекомендации по улучшению и по созданию упражнений?»* – и копирует в буфер обмена.
- Не нужно автоматически отправлять на DeepSeek через API – только ручное копирование пользователем.

### 12. UI и UX
- Адаптивный дизайн (мобильные, планшеты, десктоп). Крупные кнопки для сенсорного ввода.
- Кнопки выбора ступени (1–8) – крупные, с цветовой кодировкой; под номером ступени можно показывать ноты аккорда.
- **Обозначение нот в UI** (настройка, два режима на выбор пользователя):
  1. **Буквенные имена** (C, D, E …) с диезами или бемолями;
  2. **Ступенные цифры** (1, 2, 3 …) с диезами или бемолями для хроматических звуков.
  - В обоих режимах знаки альтерации выбираются **по тональности (ключу)**: в тональностях с диезами в ключе — писать диезы; с бемолями — бемоли; для до мажора / ля минора без знаков — без альтераций, где возможно. Энгармоника не смешивается (не писать одновременно F# и Gb в одной тональности).
- Громкость регулируется стандартным ползунком.
- Кнопки «Повторить с каденцией» и «Только вопрос» (аккорд без каденции).
- Отображение текущей гаммы и тональности на экране.
- Возможность прервать упражнение и начать заново.
- Кнопка **«Пропустить вопрос»** при затруднении (см. §5).
- Тёмная/светлая тема (опционально).

### 13. Приоритеты разработки (MVP)
**Фаза 1 (обязательно):**
- Звук (фортепиано, аккорды блоком, темп по умолчанию 90 bpm)
- Стандартные мажорные гаммы, выбор гаммы, выбор ступеней (1–7 без 8-й), количество вопросов
- Каденция T–D–S–T (фиксированная), последовательности после правильного ответа (1..7)
- Статистика за текущую сессию, цветовая подсветка по умолчанию
- Сохранение упражнений в IndexedDB
- Регистрация/логин (простой email+password) + локальная статистика

**Фаза 2:**
- 8-я ступень, диапазон октав, возможность выбора октав
- Пользовательские гаммы (свой набор нот, автоматическое построение аккордов по терциям)
- Адаптивная система (правила 1 и 2)
- Выбор типа каденции
- Экспорт статистики + копирование промпта для DeepSeek
- Регулировка темпа

**Фаза 3 (расширения):**
- Назначение разных инструментов для нот (до-фортепиано, ми-скрипка...)
- Гармонический/мелодический минор
- Синхронизация с сервером полная
- Гранулярная настройка цветовых порогов

### 14. Технологический стек
- **Frontend:** Vue 3 + TypeScript
- **Сборка / PWA:** Vite + `vite-plugin-pwa` (Workbox)
- **Хранение:** IndexedDB через Dexie.js
- **Звук:** Web Audio API + утилиты для построения аккордов и голосоведения
- **Бэкенд для регистрации и синхронизации (фаза 2+):** Firebase Auth + Firestore или Supabase — выбрать при реализации облака

### 15. Принятые решения (зафиксировано)
| Тема | Решение |
|------|---------|
| Паузы и темп | Паузы **0,5 с**, фиксированные; длительность аккордов от темпа |
| Октавы вне диапазона | Такие ступени **не задаются** в вопросах |
| Стек | **Vue 3 + TS**, Vite PWA |
| Неверный ответ | Ошибка сразу; остаёмся на вопросе до верного ответа или «Пропустить» |
| Голосоведение | Максимально естественное (см. §1) |
| Ноты в UI | Буквы **или** ступенные цифры; диез/бемоль по знакам ключа |
| Bigram-матрица | MVP — в сессии; позже — глобально, последние ~100 переходов |
| Одна ступень в тренировке | Адаптивность не нужна — всегда эта ступень |
| Bigram и prev | Учитывать только если на `prev` в итоге ответили верно |
| Процент упражнения | По **вопросам**: верно, если в итоге угадали (после ошибок — ок); пропуск — неверно |
| Подсказка после ошибки | Только при настройке; иначе ждём верный ввод |
