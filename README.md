# StepEar

**Слуховой тренажёр «Угадай ступень»** — прогрессивное веб-приложение (PWA) для тренировки распознавания ступеней мажорной гаммы по звучанию аккордов.

После вступительной каденции вы слышите аккорд одной из ступеней и выбираете её номер (1–8). После верного ответа (с учётом правил зачёта) может проигрываться **закрепляющая последовательность**. Приложение работает в браузере, устанавливается на экран телефона как PWA и сохраняет данные локально.

Полная продуктовая спецификация: [spec.md](./spec.md).

## Как это работает

### Цикл одного вопроса

1. **Каденция** по умолчанию **T–S–D–T** (ступени **1 → 4 → 5 → 1**): тоника → субдоминанта → доминанта → тоника.
2. Пауза **0,5 с** (фиксированная).
3. **Аккорд-вопрос** — одна из ступеней, выбранных для тренировки и доступных в заданных октавах.
4. Вы нажимаете кнопку ступени на сетке **1–8**.
5. При **верном** ответе (см. [зачёт](#статистика-и-зачёт-ответов)) — при необходимости **закрепление**, затем следующий вопрос.
6. При **ошибке** — остаётесь на том же вопросе; к следующему переходите только после правильного ответа или «Пропустить».

### Закрепление после правильного ответа

| Ступень вопроса | Направление | Последовательность |
|-----------------|-------------|-------------------|
| 1 | — | *(не играется)* |
| 2 | вниз | 2 → 1 |
| 3 | вниз | 3 → 2 → 1 |
| 4 | вниз | 4 → 3 → 2 → 1 |
| 5 | вверх | 5 → 6 → 7 → 8 |
| 6 | вверх | 6 → 7 → 8 |
| 7 | вверх | 7 → 8 |
| 8 | — | *(не играется)* |

В закреплении всегда звучит **полная педагогическая цепочка** (например, 5→6→7→8), даже если ступени 6 и 7 не выбраны для вопросов. Настройки «какие ступени тренировать» относятся **только к вопросам**, не к закреплению.

### Интерфейс ступеней

- На экране тренировки отображаются **все 8 ступеней** в **одном ряду** (удобно в портретной ориентации PWA).
- Ступени, **недоступные для вопросов** (не выбраны в упражнении или не помещаются в октавы), видны **приглушёнными** — на них нельзя нажать, ответ не фиксируется.
- При проигрывании **каденции и закрепления** соответствующие кнопки **подсвечиваются** (визуальное сопровождение).
- Опция **«Подсвечивать ступень при проигрывании вопроса»** (по умолчанию выкл.) — отдельно от каденции/закрепления.

## Возможности

### Реализовано

| Область | Описание |
|--------|----------|
| **Тональности** | 12 мажорных гамм или **случайная** тональность на каждый вопрос |
| **Ступени** | Вопросы по выбранным ступеням 1–8; 8-я — тоника октавой выше |
| **Октавы** | Диапазон фортепиано A0–C8; по умолчанию первая октава (C4–B4) |
| **Инструменты** | Фортепиано и классическая гитара (нейлон), семплы MP3 |
| **Голосоведение** | Обращения и связность регистра между аккордами вопроса |
| **Темп** | 40–160 BPM; длительность аккордов от темпа, паузы 0,5 с |
| **Распределение вопросов** | Равномерный план по выбранным ступеням (`balancedSteps`) |
| **Повторы** | «Повторить с каденцией», «Только вопрос», «Пропустить», «Прервать» |
| **Подсказка** | Показ правильной ступени после ошибки (опционально) |
| **Статистика сессии** | Процент, списки «без ошибок» / «с ошибкой», разбивка по ступеням |
| **Пресеты** | Сохранение и загрузка упражнений в IndexedDB |
| **Аккаунт** | Локальная регистрация email + пароль (SHA-256), без облака |
| **PWA** | Офлайн-кэш, установка на домашний экран |
| **Обозначение нот** | Буквы (C, D, E…) или ступенные цифры; знаки по тональности |

### Планируется (см. [spec.md](./spec.md))

- Выбор типа каденции (несколько вариантов)
- Адаптивная подборка вопросов (слабые ступени, bigram)
- Пользовательские гаммы, минорные лады
- Экспорт статистики и промпт для ИИ
- Синхронизация с сервером
- Назначение разных инструментов на ноты

## Статистика и зачёт ответов

- **Процент сессии** считается **по вопросам**, не по числу нажатий.
- Вопрос засчитывается **верным**, только если правильная ступень выбрана **без ошибочных попыток** на этом вопросе.
- После ошибки можно продолжать отвечать и перейти дальше только с **верным** ответом, но вопрос в зачёт **не войдёт**.
- **«Пропустить»** — вопрос завершён **без зачёта**.
- Если в упражнении выбраны **и 1-я, и 8-я** ступени, ответы **1** и **8** **не** взаимозаменяемы — нужно угадать именно сыгранную ступень.
- Если выбрана только одна тоника (1 или 8), **1** и **8** по-прежнему могут считаться эквивалентными при ответе (см. `src/music/tonic-steps.ts`).

## Быстрый старт

```bash
git clone <url-репозитория>
cd StepEar
npm install
npm run dev
```

Откройте в браузере адрес из вывода Vite (обычно [http://localhost:5173](http://localhost:5173)).

1. **Настроить упражнение** — `/setup` (тональность, ступени, октавы, темп, число вопросов).
2. **Начать тренировку** — кнопка на экране настройки или **Быстрый старт** на главной (`/train` с параметрами по умолчанию).

## Публикация в подпапке (например `/stepear/`)

1. Скопируйте `.env.example` в `.env.production` и задайте путь:

```env
VITE_BASE_PATH=/stepear/
```

2. Соберите: `npm run build`.

3. Загрузите **содержимое `dist/`** в каталог `stepear` на сервере (URL: `https://site_name.ru/stepear/`).

4. Настройте веб-сервер: для любого пути внутри `/stepear/` отдавать `stepear/index.html` (SPA fallback).

**Nginx (пример):**

```nginx
location /stepear/ {
  alias /var/www/html/stepear/;
  try_files $uri $uri/ /stepear/index.html;
}
```

Для **корня домена** оставьте `VITE_BASE_PATH=/` или не создавайте `.env.production`.

Локальная проверка подпапки:

```bash
# PowerShell
$env:VITE_BASE_PATH="/stepear/"; npm run build; npm run preview
```

## Установка PWA

1. Соберите проект: `npm run build`.
2. Разместите `dist/` на HTTPS-хостинге **или** используйте `npm run preview` для проверки.
3. В Chrome / Edge / Safari: «Установить приложение» / «На экран Домой».

Сервис-воркер кэширует JS, CSS, HTML и **MP3-семплы** для работы без сети после первой загрузки.

## Звук и семплы

- Семплы **MusyngKite** (лицензия MIT) в каталогах:
  - `public/samples/piano/` — фортепиано
  - `public/samples/guitar/` — классическая гитара (нейлон)
- По одному файлу на ноту **четвёртой октавы** (C4–B4); остальная высота — **сдвиг pitch** в Web Audio API.
- Аккорды играются **блоком** (одновременно).
- **Тоника** строится в **нижней** выбранной октаве; корень ступени — в своей октаве гаммы; верхние звуки аккорда могут переноситься выше при необходимости.
- **8-я ступень** в аудио — тоника **на октаву выше** (отдельно от 1-й), см. `src/music/step8.ts`.

## Настройки упражнения

Настраиваются на экране **«Настройка упражнения»** (`/setup`):

| Параметр | Описание |
|----------|----------|
| Тональность | 12 мажорных или случайная на каждый вопрос |
| Инструмент | Фортепиано / гитара |
| Октавы | Какие октавы фортепиано доступны; от этого зависит, какие ступени **можно задавать в вопросах** |
| Ступени для вопросов | Чекбоксы 1–8 |
| Равномерное распределение | Примерно одинаковое число вопросов на каждую выбранную ступень |
| Число вопросов | 1–200 (например, 20) |
| Темп (BPM) | 40–160 |
| Подсказка после ошибки | Показать правильную ступень |
| Подсветка вопроса при проигрывании | По умолчанию **выкл.** |
| Обозначение нот | Буквы или ступенные цифры (глобально в настройках) |
| Громкость | Ползунок |

Пресеты сохраняются в IndexedDB и загружаются с экрана **«Сохранённые упражнения»** (`/saved`).
